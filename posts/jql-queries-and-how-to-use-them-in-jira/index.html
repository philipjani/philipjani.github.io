Why Do Scrum Masters Need to Know JQL?

Within a real world Jira project, there could easily be thousands, if not tens of thousands of individual issues across hundreds of projects and boards that need to be tracked across a variety of inflight statuses. Jira Queries can also be saved as JQL filters and used to create dashboards that create added transparency for team members, senior leadership or stakeholders.

In order to make this task of tracking issues easier, Jira has a built in search mechanism called “JQL” which stands for Jira Query Langauge and in many ways it is similar to SQL (Structured Query Langauge).

What is JQL?

JQL is a way to search for any issue in Jira by it’s particular details. Examples include searching for an issue based on it’s particular title (or summary as it is referred to in Jira), by the Epic it is associated to, what team is working on it or what sprint it was worked on. Any detail that is tracked within the issue can be used to perform a JQL search and return that issue or a host of related issues.

How to use JQL within Jira

At the top of a Jira Software instance, you will see a blue banner that says, “Dashboards”, “Projects”, “Issues”, “Boards”, “Plans”, etc. In order to access the JQL search bar, you will need to click on “Issues” which will bring down a drop down menu with many different options. What you want to click on is, “Search for issues” within this dropdown menu. This will bring out the “Search” bar and this is where you will want to input your JQL query.

Logical Operators

Similar to SQL queries, JQL uses logical operators to create more complex queries and data filters with additional logic. The most common ones would be “AND”, “OR” and “NOT”.

To Find All 'Open' Issues:
Team = 544 AND Sprint in openSprints() AND status not in (Blocked, Delivered, Accepted, Closed) ORDER BY priority DESC
Logical Operator Returns
AND Returns True if both logical statements return True.
OR Returns True if either logical statement returns True or both logical statements return True.
NOT Returns True if logical statement returns False.

JQL Examples:

Finding Child Issues (Stories, Defects) of an Epic:

Example:
issue in childIssuesOf(“MP-6984”)

Search for Issues without Fix Versions:

Example:
fixVersion = empty

Search Issues from a Specific Sprint:

Example:

sprint in “<sprint_name>”

team = 544 and sprint in “Roku 22.19”

Accepted Issues in the Past 3 months with no Fix Versions

Example:
team = 544 AND fixversion = EMPTY AND createdDate >= startOfMonth(-3) AND status = accepted

All Issues in Currently Active Sprint

Example:
team = 544 AND Sprint in openSprints() ORDER BY updated DESC

Defects Created Today

Example:
team = 544 AND issuetype = defect AND createdDate >= startOfDay()

Issues without Fix Versions in Current Sprint

Example:
team = 544 AND fixversion = EMPTY AND sprint in openSprints()

Accepted Issues

Example:
team = 544 AND status in (Accepted) ORDER BY priority DESC

Blocked Issues

Example:
team = 544 AND status in (Blocked) ORDER BY priority DESC

Closed Issues (past 7 days)

Example:
team = 544 AND status = closed AND updated >= -7d

Defects in Current Sprint

Example:
team = 544 AND issuetype = Defect AND Sprint in openSprints()

Epics

Example:
issuetype = epic AND team = 544 AND status in (new, “In Progress”) ORDER BY priority DESC

Open Issues

Example:
team = 544 AND Sprint in openSprints() AND status not in (Blocked, Delivered, Accepted, Closed) ORDER BY priority DESC

Saving JQL Queries as JQL Filters

Once you have entered a JQL search string into the appropriate search box, Jira will populate a “Save as” button next to the Search title above the search box. If you want to save your query as a filter for easy reference later, you can click “Save as” and then enter the “Filter Name” in the appropriate field that pops up.

You can find your saved filters to the right of the search box in a drop down menu under “Favorite Filters”. These filters can be used to populate dashboards and confluence pages with the returned data.
